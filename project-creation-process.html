How the Geoportal Was Created

The development of the CDNGI Geoportal followed a structured process designed to implement a fully functional Spatial Data Infrastructure (SDI) for Cape Town. The process ensured that spatial datasets were not only accessible, but also usable, interoperable, and standards-compliant.

  1. Data Collection & Preparation: Datasets including aerial imagery, DEMs, cadastral boundaries, and control points were sourced, cleaned, and stored in a spatially enabled database hosted on QGIS Cloud.
  2. GeoServer & WMS/WFS Configuration: Using QGIS Cloud's built-in GeoServer, datasets were published as WMS (Web Map Services) and WFS (Web Feature Services) for consumption in GIS applications and the web.
  3. Metadata Catalogue: A metadata system was designed to describe datasets using standardized fields such as title, summary, keywords, coordinate reference system, and source.
  4. Interactive Web Map: The front-end map was developed using Leaflet.js, enabling users to explore spatial layers interactively. Features like index grids and clickable tiles allow users to access dataset metadata and download links.
  5. Geoportal Integration: All components—map viewer, metadata catalogue, and download functionality—were integrated into a single cohesive platform with a clean, accessible user interface built with HTML and Tailwind CSS.

The result is a modern, open-access Geoportal supporting spatial data discovery, visualization, and download—custom-built for the case study area and fully aligned with SDI principles.
